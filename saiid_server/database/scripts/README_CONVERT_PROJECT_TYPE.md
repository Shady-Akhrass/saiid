# 📘 دليل تحويل project_type من ENUM إلى VARCHAR

## المشكلة

عند محاولة إنشاء مشروع بنوع مشروع جديد (مثل "الكفالات")، يظهر الخطأ التالي:
```
SQLSTATE[01000]: Warning: 1265 Data truncated for column 'project_type'
```

## السبب

عمود `project_type` في قاعدة البيانات هو `ENUM` بقيم محددة فقط:
- `['إغاثي', 'تنموي', 'طبي', 'تعليمي']`

لكن أنواع المشاريع الآن ديناميكية - يتم إدخالها يدوياً من قبل الأدمن وتظهر في القائمة.

## الحل: تحويل ENUM إلى VARCHAR

### الخطوة 1: فتح phpMyAdmin

1. افتح phpMyAdmin
2. اختر قاعدة البيانات الخاصة بالمشروع
3. اضغط على تبويب "SQL"

### الخطوة 2: نسخ ولصق SQL Script

1. افتح الملف: `database/scripts/convert_project_type_to_varchar.sql`
2. انسخ المحتوى بالكامل
3. الصقه في phpMyAdmin
4. اضغط "Go" أو "تنفيذ"

### الخطوة 3: التحقق من النجاح

بعد التنفيذ، يجب أن ترى رسالة:
```
Migration completed successfully!
new_column_type: varchar(255)
```

بعد التحويل:
- ✅ يمكن إضافة أي نوع مشروع جديد
- ✅ لا توجد قيود على القيم
- ✅ أنواع المشاريع ديناميكية تماماً

## إذا استمرت المشكلة

1. **تحقق من قاعدة البيانات:**
   ```sql
   DESCRIBE project_proposals;
   ```
   يجب أن ترى `project_type` كـ `varchar(255)` وليس `enum(...)`.

2. **تحقق من الأخطاء:**
   - افتح `storage/logs/laravel.log`
   - ابحث عن أخطاء SQL

3. **مسح Cache:**
   ```bash
   php artisan cache:clear
   php artisan config:clear
   ```

## نهاية الدليل

بعد تطبيق SQL Script، يجب أن يعمل إنشاء المشاريع بأي نوع مشروع بشكل صحيح.
